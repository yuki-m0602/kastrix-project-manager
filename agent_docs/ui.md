# UI実装ガイド

## モックアップの扱い

モックアップは `mockup/` に格納されている。
実装前に必ずモックアップ全体を確認してから着手する（部分だけ見て実装しない）。

確認すべき項目：
- 全体のレイアウト構成
- コンポーネントの境界
- インタラクション（hover, active, 遷移など）
- モックアップに明示されていない状態（後述）

---

## 忠実度の方針

モックアップへの忠実度は「雰囲気重視」ではなく「可能な限り忠実に」を目指す。
ただし技術的制約がある場合は自己判断せず、ユーザーに報告して判断を仰ぐ。

---

## CSSで再現困難な箇所の対応

角丸・曲線・独自シェイプはCSSの `border-radius` では再現できないケースがある。

対応方針（優先順位順）：
1. SVGで実装する
2. モックアップから画像として切り出して使用する
3. CSSで近似できる場合は近似案をユーザーに提示して承認を得る

**再現困難と判断した場合は自己判断で妥協せず、必ずユーザーに報告する。**

報告フォーマット：
```
【再現困難な箇所】
（該当箇所の説明）

【理由】
（なぜCSSで再現できないか）

【代替案】
（SVG実装 / 画像切り出し / CSS近似案）
```

---

## モックアップにない状態の扱い

モックアップはデザインの一状態しか示していないことが多い。
以下の状態についてはユーザーに確認してから実装する：

- ローディング状態
- エラー状態
- 空状態（データなし）
- レスポンシブ対応（モックアップがPC想定の場合）

---

## コンポーネント設計

- 分割粒度はプロジェクトの既存コンポーネントの粒度に合わせる
- 新しい分割方針が必要な場合は実装前にユーザーに提案する
- 状態管理（ローカルstateにするか外に出すか）は既存パターンに合わせる

---

## モックアップとのズレが生じた場合

実装後にモックアップとズレがある場合は、自己判断で修正しない。
ズレの箇所・理由・修正案をユーザーに提示して判断を仰ぐ。
